End-to-End Benchmark

Can AI generate a quantity schedule from construction drawings?

TakeoffBench is the first benchmark measuring AI's ability to perform end-to-end quantity takeoff—producing structured output that estimators can actually use.

The Task

End-to-End Quantity Takeoff

Unlike component benchmarks that test individual capabilities (detection, counting), TakeoffBench evaluates the complete pipeline that estimators actually need.

  • Identify all elements on the drawing
  • Classify elements correctly (door type, window size)
  • Count discrete items, measure linear/area quantities
  • Map elements to CSI MasterFormat divisions
  • Output structured JSON matching our schema
Why End-to-End Matters
Component benchmarks "Found 12 doors" ✓
Missing context What type? What size?
Not usable Estimator can't bid with this
TakeoffBench Full schedule, ready to price
Evaluation

Structured Metrics

Objective evaluation comparing predicted schedules against professional estimator output.

40% weight
Element Recall
Did the model find all elements that should appear in the schedule?
30% weight
Quantity Accuracy
For elements found, are counts and measurements correct? (MAPE)
20% weight
Classification
Are element types, sizes, and specifications correct?
10% weight
CSI Mapping
Are items assigned to correct MasterFormat divisions?

Output Format

Structured Schema

Models must output JSON matching our schema for objective evaluation.

Output Schema

{
  "project_id": "string",
  "takeoff": {
    "[CSI Division]": {
      "[CSI Code]": [
        {
          "description": "string",
          "quantity": number,
          "unit": "EA" | "LF" | "SF" | "CY",
          "specifications": {}
        }
      ]
    }
  }
}

Example Output

{
  "project_id": "residential_042",
  "takeoff": {
    "08 - Openings": {
      "08 14 16 - Wood Doors": [
        {
          "description": "3'-0\"×6'-8\" HC Door",
          "quantity": 6,
          "unit": "EA"
        }
      ]
    }
  }
}
Dataset

Real Construction Drawings

Ground truth from professional estimators, not synthetic annotations.

Drawing Sources

Architectural floor plans from real residential and commercial projects. Mix of CAD exports and scanned documents with varying quality levels.

Ground Truth

Each drawing annotated by professional estimators with 5+ years experience. Multiple annotators per drawing for inter-rater reliability.

Element Types

Doors, windows, walls, rooms, fixtures, cabinets, appliances, and specialty items. Both count-based and measurement-based quantities.

Held-Out Test Set

Test drawings are never released. Models submit predictions to our evaluation server to prevent overfitting.

Quick Start

Get Started

Evaluate your model in four steps.

# Install
git clone https://github.com/Jp3stephens/takeoffbench
cd takeoffbench && pip install -e .

# Download validation set
takeoffbench download --split val

# Run your model (example with baseline)
python evaluate.py \
  --model claude-3.5-sonnet \
  --input data/val/ \
  --output predictions.json

# Score locally
takeoffbench score --predictions predictions.json --split val

# Submit to test leaderboard
takeoffbench submit --predictions predictions.json --model-name "Your Model"
